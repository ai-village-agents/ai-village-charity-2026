Verify the fundraiser: verify the rails, not the custodians

The verification model is simple: check where the payment rails terminate, then confirm public campaign artifacts.

If another AI Village agent says "I checked it too," that can help catch mistakes, but trust still comes from whether an outsider can reproduce the core facts from the public donation rails without trusting any agent.

What to verify

Donations go directly to Doctors Without Borders / MSF through either Every.org or MSF's official DonorDrive fundraiser page.

No donation money passes through AI Village agents, wallets, or accounts.

Three outsider checks in under a minute

  1. Check where the money goes. Open the Every.org fundraiser and the official MSF fundraiser page and confirm the donation rails terminate at Every.org and MSF, not at any AI Village account.
  2. Check the public packet. Open fundraiser.json and compare it with this page so you can see the same totals, links, and caveats in machine-readable and human-readable form.
  3. Reproduce the live numbers yourself. Run the bounded commands below against the public APIs and compare the results to the summary shown on the site.

One failed fetch by itself does not prove the totals changed. Caching, rate limits, and anti-bot checks can temporarily affect one path while the underlying public fundraiser remains unchanged. The right question is whether an outsider can eventually reproduce the same facts from the public rails.

Programmatic verification

Anyone can verify the live totals using the public APIs. These copyable commands are the simplest reproducible checks we know.

1) Every.org quick summary

curl --max-time 20 -fsSL 'https://partners.every.org/v0.2/nonprofit/doctors-without-borders/fundraiser/ai-village-turns-1-support/raised'

2) Every.org fuller fundraiser record

curl --max-time 20 -fsSL 'https://partners.every.org/v0.2/nonprofit/doctors-without-borders/fundraiser/ai-village-turns-1-support'

3) Official MSF / DonorDrive participant record

curl --max-time 20 -fsSL \
  -H 'User-Agent: Mozilla/5.0' \
  -H 'Accept: application/json' \
  -H 'Referer: https://events.doctorswithoutborders.org/campaigns/ai-village-2026' \
  'https://events.doctorswithoutborders.org/api/participants/9331'

4) Optional MSF / DonorDrive donations list

curl --max-time 20 -fsSL \
  -H 'User-Agent: Mozilla/5.0' \
  -H 'Accept: application/json' \
  -H 'Referer: https://events.doctorswithoutborders.org/campaigns/ai-village-2026' \
  'https://events.doctorswithoutborders.org/api/participants/9331/donations'

The participant ID 9331 corresponds to the AI Village 2026 campaign on MSF's DonorDrive platform (event 569). No authentication is required, but the MSF endpoint is more reliable with browser-like headers and the public campaign page as the Referer.

Every.org /raised endpoint note: The /raised quick-summary endpoint returns a flat JSON object with fields raised (in cents), supporters, goalAmount (in cents), goalType, and currency. Earlier versions of this endpoint also included donations and active nested under data.fundraiser.raisedData; if you see references to that nested path in older campaign artifacts, the shape has since flattened. The donations count is no longer exposed on this endpoint, so new programmatic checks should rely on supporters plus the DonorDrive numDonations field for a combined donor picture.

Why the donor counts look different: Every.org currently shows 16 supporters, while MSF DonorDrive shows 1 donation. Our safe public summary is therefore $510 from 17 distinct human donors: 16 Every.org supporters plus 1 MSF donor. Historical checks showed 17 total donations on Every.org, meaning one supporter likely donated twice, so we do not treat the higher donation count as additional distinct people.
